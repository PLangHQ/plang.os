# plangOS v1 — Process Lockdown

The image is built to be safe even when run with default Docker settings, and
**much** safer when run with the recommended runtime flags below. This file
documents both layers.

## Image-level (baked in)

| Control | Value | Enforced by |
|---|---|---|
| PID 1 | tini v0.19.0 static, SHA-256 verified | `ENTRYPOINT ["/usr/local/bin/tini", "--", "/opt/plang/plang"]` |
| User | `plang` UID `10001`, GID `10001` | `USER 10001:10001` in Containerfile |
| Shell | **absent** | Final `RUN` removes `/bin/sh`, `/bin/ash`, `/bin/bash`, `/bin/login`, `/bin/su`, and `/bin/busybox` itself |
| Package manager | **absent** | Final `RUN` removes `/sbin/apk`, `/etc/apk`, `/usr/sbin/apk`, `/var/cache/apk`, `/lib/apk` |
| Setuid/setgid | **none** | `build.sh` audit fails if any remain after image assembly |
| Home dir | `/home/plang`, owned by `10001:10001` | `COPY --chown=10001:10001` from staging |
| `/etc/passwd` | two lines (root + plang, both `/sbin/nologin`) | Staged file, not generated by `adduser` |
| Base images | pinned by SHA-256 digest | `build.sh` resolves tag → digest at build, passes via `--build-arg` |
| Globalization | invariant mode (no ICU) | `DOTNET_SYSTEM_GLOBALIZATION_INVARIANT=1`, and zips published with `-p:InvariantGlobalization=true` |
| Telemetry | off | `DOTNET_CLI_TELEMETRY_OPTOUT=1`, `DOTNET_NOLOGO=1` |
| Diagnostics | off | `DOTNET_EnableDiagnostics=0` (closes the .NET diag port) |

## Runtime flags (documented, enforced by caller)

```sh
docker run --rm \
  --read-only \
  --cap-drop=ALL \
  --security-opt=no-new-privileges \
  --security-opt=seccomp=default.json \  # docker default; v2 will ship a custom profile
  --tmpfs /tmp:rw,noexec,nosuid,size=64m \
  --user 10001:10001 \
  --pids-limit=64 \
  --memory=512m --memory-swap=512m \
  plang-os:<tag> <plang args>
```

| Flag | Why |
|---|---|
| `--read-only` | Rootfs is immutable at runtime. PLang only needs `/tmp` (tmpfs) and whatever data volume the caller mounts. |
| `--cap-drop=ALL` | PLang has no known need for any Linux capability in container mode. If a future module does (e.g. `CAP_NET_BIND_SERVICE` for port < 1024), it will be added explicitly and justified. |
| `--security-opt=no-new-privileges` | Prevents any path by which a child process could gain privileges via setuid. Belt-and-braces since the image has no setuid binaries anyway. |
| `--tmpfs /tmp:noexec,nosuid` | PLang may write scratch data to `/tmp`. `noexec` blocks executing anything dropped there; `nosuid` ignores any setuid bit. |
| `--user 10001:10001` | Redundant with image's `USER` directive but defends against the image being retagged with a different default. |
| `--pids-limit=64` | Caps PID exhaustion. PLang is single-process + tini + a handful of .NET threads — 64 is generous. |
| `--memory / --memory-swap` | Hard cap. Exact values depend on workload; set deliberately. |

## Seccomp

v1 ships with **default Docker seccomp only**. That profile already blocks
~44 syscalls including `reboot`, `kexec_load`, `bpf`, `perf_event_open`,
`pivot_root`, `mount`, `umount2`.

v2 will ship a tighter profile derived from `strace -c -f plang --test …` on a
real workload, then curated down to the syscalls PLang actually issues. The
delta (default profile vs a PLang-specific profile) is where the v2 security
gain comes from.

## AppArmor / SELinux

Not configured in v1. Both are host-distro-specific; we can't ship a profile
that applies uniformly across hosts. A future version will publish:
- an AppArmor profile for Debian/Ubuntu hosts
- an SELinux policy module for RHEL/Fedora hosts

## Network

Image defines no `EXPOSE`. No listen sockets. If a PLang goal wants to serve
HTTP, the caller binds a port at runtime via `-p`. See `attack-surface.md` for
the current listen-port count (expected: zero).

## Secrets

No secrets are baked into the image. PLang settings, LLM API keys, and the
data directory must be mounted at runtime:

```sh
-v "$HOME/.plang:/home/plang/.plang:ro"
-v "$PWD:/workspace:ro"
-e "PLANG_LLM_API_KEY=${PLANG_LLM_API_KEY}"
```

## Boot-path integrity

Not applicable in v1 (container, not bootable). dm-verity + Secure Boot + UKI
come with the appliance artifact.

## Signals

- `SIGTERM` / `SIGINT` → tini forwards to plang → .NET handles SIGTERM via
  `AppDomain.ProcessExit` / `Console.CancelKeyPress`. Expect clean shutdown
  within 5 s.
- `SIGKILL` → container dies immediately; data-dir consistency is PLang's
  concern.
- `SIGCHLD` → tini reaps. PLang should never spawn children itself in v1; if
  it does (future PLang module shelling out), tini catches the zombies.
